# Code Of Conduct

Killer Queen should be an easy repo to contribute to and a hard repo to waste people’s time in.

## Expected Behavior

- be respectful
- give practical feedback
- critique the work, not the person
- credit other people’s contributions
- keep discussion focused on improving the game

## Unacceptable Behavior

- harassment or personal attacks
- bigoted language
- hostile dogpiling
- bad-faith review behavior
- spam, prompt sludge, or low-effort noise that makes collaboration harder

## Maintainer Standard

Maintainers should explain decisions clearly, close the loop on feedback, and keep the repo readable for the next contributor.

## Reporting

If someone is making the project worse to work on, contact the maintainers through a GitHub issue or direct channel with enough detail to act on it.
