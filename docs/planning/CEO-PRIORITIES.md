# CEO Priorities

## Current Truth

The game has crossed the line from horizontal mission prototype to vertical tower prototype.

That is the right move.

Do not drift backward into:

- pickup missions
- extraction-pad objectives
- combat-first design
- content expansion before feel is proven

The only thing that matters right now is whether the climb loop is addictive.

## What "Good" Looks Like

For milestone 1, the game is good if a player says:

- "I almost missed that, but I saved it."
- "I see a faster route next run."
- "I want another attempt immediately."

If they instead say:

- "I guess I beat it."
- "The movement is fine."
- "The jetpack is useful."

then the game is not there yet.

## Do This Next

In this exact order:

1. Lock the first 20 seconds of the run.
2. Tune movement until jump, wall kick, burst, and spring all feel meaningfully different.
3. Tune the tower route so it creates one early recovery, one optional detour, and one obvious "next time I go that way" moment.
4. Tune the kill floor so it creates urgency without feeling unfair.
5. Only then polish HUD copy and visual readability.

## First 20 Seconds Checklist

The opening stretch must prove the game.

- one safe opening platform
- one spring or burst save moment
- one wall contact that naturally teaches wall kick timing
- one small risk-reward detour for fuel or route advantage
- one hazard that pressures movement instead of stopping it

If the first 20 seconds are weak, the rest of the tower does not matter yet.

## Tuning Order

### Movement

- wall slide should buy decision time, not stall forever
- wall kick should feel reliable, not strict
- burst should feel like a save, not a float
- jump and wall kick should not blur together

### Tower

- route lines should alternate left and right enough to create rhythm
- vertical walls should support intentional parkour, not accidental sticking
- crumble platforms should force movement, not feel random
- spring pads should create delight, not chaos

### Pressure

- drones should create timing pressure
- mines should create route pressure
- the kill floor should punish hesitation, not experimentation

### HUD

- the player should always know:
  - how high they are
  - whether they have movement resources
  - whether they are on a chain
- remove any wording that sounds like a mission objective

## Hard No List

Do not work on these until the climb loop is undeniably fun:

- endless generation
- more enemy types beyond what is needed for pressure
- multiple weapons
- story framing
- progression systems
- menus beyond what the run needs
- "juice" that hides weak movement

## Quality Bar

This project should feel premium through:

- movement clarity
- route readability
- sharp restart cadence
- strong silhouettes
- restraint

Not through:

- more systems
- more pickups
- more copy
- more particles
- more clutter

## Next Art Move

Once the movement loop is locked, the first art pass should be:

- player state silhouettes
- platform readability
- hazard weak-point readability
- one scrapyard background layer with strong depth

That is enough to move the game from prototype to identity.

## Team Rule

If two people are working at once:

- one person touches feel and runtime
- one person touches product direction, tuning order, and presentation

Do not have both people improvising mechanics at the same time.

## Final Call

The ship direction is simple:

Make the first minute thrilling.
Make the restart instant.
Make the movement trustworthy.
Make the art readable.

Everything else waits.
