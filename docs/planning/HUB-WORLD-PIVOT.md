# HUB WORLD PIVOT

## Product Call

`Killer Queen` should pivot from open-ended metroidvania structure to a hub-world plus mission-stage structure.

The right comparison is not “copy Mario 64 literally.”

The right comparison is:

- one memorable central playground
- several distinct stages
- repeated returns to the same spaces under different mission conditions
- progression driven by marks, unlocks, and growing mastery

This fits the game better than a seamless map because the current strengths are:

- memorable movement
- strong region identity
- authored route design
- replayable mission remix potential

## Hub Thesis

The hub should be a reclaimed sanctuary above the hive breach: a royal transit court shattered by infestation and half rebuilt into a resistance outpost.

It should feel like:

- a safe zone
- a vertical parkour toy
- a map of promises
- a place that changes as you progress

The hub must not be oversized. Its purpose is legibility, personality, and return value, not scale flexing.

## Recommended Structure

- `1` compact hub world
- `6` core stages
- `10` missions per stage
- hub secrets and side marks on top

Why `6 x 10`:

- enough breadth to feel like a full game
- enough density to make each stage worth learning
- still realistic for stage identity, enemy tuning, and art cohesion

## Stage Set

1. `Hive Shaft`
- escape pressure
- collapsing routes
- aggressive verticality

2. `Sunset Ruin Skyline`
- broken towers
- long sightlines
- bridge traversal and chase routes

3. `Overgrown Foundry`
- roots in machinery
- hidden rooms
- layered lateral routes

4. `Flooded Transit`
- lift systems
- waterlogged conduits
- route timing and hazard cycling

5. `Royal Armory`
- locked halls
- elite enemy pressure
- weapon-gate mission logic

6. `Queen's Core`
- ritual machinery
- bio-mechanical climax
- hardest route and boss variants

## Mission Format

Every stage should support around ten marks that remix the same authored layout.

The missions should vary:

1. `Summit`
- reach the high point or key chamber

2. `Rush`
- beat the route under time pressure

3. `Nestbreaker`
- clear all marked combat nests

4. `Relic Hunt`
- find a hidden cache or royal relic

5. `Relay Run`
- activate nodes in sequence in one life

6. `Hazard Variant`
- rising danger, lockdowns, or alternate state of the stage

7. `Restriction Run`
- finish using a constrained rule set

8. `Elite Hunt`
- chase and kill a roaming high-value target

9. `Sub-route Unlock`
- open a locked route and finish through it

10. `Champion`
- stage boss or mission-specific champion fight

## Hub Progression Rules

- marks unlock new stage doors and upper hub sectors
- bosses or champions unlock weapon modules or traversal refinements
- hub geometry should evolve to reflect progress
- late hub secrets should reward movement mastery, not grind

## Risks To Avoid

- oversized hub with dead walking time
- missions that feel like the same route with different text
- collectible spam replacing strong movement or combat goals
- bland stage identities
- hub functioning like a static menu

## Production Recommendation

Do not try to build the full `6 x 10` structure immediately.

Build this first:

- one hub sector
- one stage
- three genuinely different missions
- one hub unlock that proves the structure

If that slice works, the format scales.
